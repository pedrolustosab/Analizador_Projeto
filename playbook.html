Playbook de Adoção

Dominando o PMO Intel Engine

Um guia tático para preparar seus dados, interpretar métricas avançadas e conduzir reuniões executivas de alto impacto utilizando o Dashboard.

⚙️ Fase 1: A Regra de Ouro (Preparando o MS Project)

O motor de inteligência do aplicativo cruza o Planejado vs. Realizado. Para que a matemática do EVM (Curva S) e os alertas de atraso funcionem, o seu arquivo do Microsoft Project precisa estar devidamente "selado".

  • Atribua Custos ou Recursos O cálculo do Valor Agregado (EV) depende de dinheiro. Certifique-se de que as tarefas possuem custos fixos ou recursos com taxas/hora cadastradas na Planilha de Recursos.
  • Ação Crítica: Salvar Linha de Base (Baseline) Antes de atualizar o progresso do projeto (%), vá na aba Projeto > Definir Linha de Base > Salvar Linha de Base. Sem isso, o sistema não saberá qual era o plano original para calcular os desvios.
  • Atualize o Progresso e a Data de Status Defina a data de status do projeto (Data de corte da análise) e atualize as colunas % Concluído e Término Real.
  • Exporte para XML Vá em Arquivo > Salvar Como e escolha o formato Dados XML (*.xml). É este arquivo que você arrastará para o nosso Dashboard.
💡

Dica de Ouro: Se o Dashboard abrir e mostrar o aviso "Análise Indisponível" na seção financeira, significa que o seu XML não possui a tag de Custo (Custos = R$ 0). Volte ao Project, adicione um custo de R$ 1.000 em uma tarefa sumária só para teste, e repita a exportação.

🧭 Fase 2: Lendo o Dashboard como um C-Level

1. Tracker de Marcos (Timeline)

No topo do dashboard, os diretores não querem ler tarefas, querem ver entregas. O Tracker filtra automaticamente todas as tarefas com Duração = 0 (Marcos).

Como interpretar:
Bolas Verdes = Entregue.
Bolas Azuis (Pulsando) = Trabalhando agora (No Prazo).
Bolas Vermelhas (Pulsando) = Trabalhando agora (Atrasado).

2. O Filtro em Cascata (Matriz)

A tabela central consolida apenas as Anomalias. Utilize o primeiro menu suspenso (Gravidade) para focar.

Fluxo Tático: Se você tem 15 minutos em uma reunião de diretoria, selecione "Somente 🔴 Incêndio". O segundo filtro se adaptará automaticamente, permitindo que você isole apenas "Marcos Rompidos" ou "Efeito Dominó".

🚨 Fase 3: Entendendo a Matriz de Causa Raiz

O motor de dados lê as entrelinhas do XML e gera 8 Tags diagnósticas (Diagnostic Analytics) exclusivas do nosso algoritmo.

Fila de Incêndio

Problemas que já estouraram a linha de base e impactam diretamente a entrega.

T2: Estouro Real T7: Marco Rompido T8: Efeito Dominó (Gargalo de Predecessora)

Fila de Atenção

Riscos iminentes. A linha final de base ainda não passou, mas o projeto está doente.

T1: Inércia (Início passou e tem 0%) T3: Desvio Projetado T6: Alerta de Ritmo (% menor que os dias passados)

Auditoria / Post-Mortem

Anomalias de comportamento da equipe ou maquiagem de cronograma.

T4: Maquiado (0%, mas empurrado pra frente) T5: Absorvido (Entregue atrasado, já resolvido)

🔍 Fase 4: O Modal de Drilldown e Exportação

Sempre que um gestor questionar um número no Gantt WBS ou na Tabela de Atrasos, basta Clicar em cima da linha da tarefa.

O aplicativo não recarregará a página. Um Modal Popup (Tela sobreposta) abrirá instantaneamente mostrando:

  • Datas exatas (Baseline vs. Real).
  • Os dias de variação matemática.
  • Drill-down: A lista de todas as subtarefas contidas naquela fase com seus percentuais de avanço.

A Mágica da Exportação Offline 🪄

No topo direito do Dashboard, clique no botão escuro "📥 Baixar Relatório HTML". O sistema empacotará todo o JavaScript, o CSS, a renderização SVG da Curva S e o Banco de Dados (JSON) em um único arquivo HTML leve.

Você pode enviar esse arquivo via e-mail ou WhatsApp para o Presidente da empresa. Ele abrirá no celular ou no computador instantaneamente, de forma interativa, sem precisar de internet, sem precisar instalar Python e sem precisar de licença do MS Project.
